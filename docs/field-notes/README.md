# 现场记录与经验沉淀

这个目录用于沉淀真实站点验证、风控触发、工具使用约束、页面结构观察结果与后续验证策略。

目标不是重复 `README.md` 或 `docs/project-prompt.md` 中的设计原则，而是补充：

- 真实验证过程中已经发生过什么
- 哪些判断是基于事实，哪些只是推断
- 某类站点为什么会触发问题
- 后续回归时应遵守什么更具体的操作顺序

## 使用原则

1. 这里优先记录“真实发生过的操作与结果”，避免事后美化。
2. 如果某个结论只是推断，应明确标注“推断”而不是写成既成事实。
3. 页面级经验优先沉淀为“可执行规则”，方便后续给 AI 或人工验证直接复用。
4. 如果某个站点已经触发风控，后续文档应先记录“停止条件”和“禁操作项”，再记录优化建议。
